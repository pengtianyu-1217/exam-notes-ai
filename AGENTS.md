# 备考助手

## 项目概述

### 目标
帮助学生快速整理教材笔记、梳理知识架构、生成配套练习题。

### 核心功能
1. 上传多种格式文档（PDF/PPT/TXT/MD/DOCX/RTF）
2. AI 智能解析章节结构（支持 Markdown 标题、编号章节、智能分割三种模式）
3. 基于备考时点生成不同深度的知识梳理
4. 支持多种学习方法选择（费曼、康奈尔、思维导图、记忆宫殿、案例分析）
5. 支持多种字体选择
6. 配套练习题生成
7. PDF 导出
8. 音频上传和语音转文字

## 技术栈

- **框架**: Next.js 16 (App Router)
- **核心**: React 19
- **语言**: TypeScript 5
- **UI 组件**: shadcn/ui
- **样式**: Tailwind CSS 4
- **字体**: next/font (Inter, Noto Serif SC, ZCOOL XiaoWei)
- **PDF 解析**: pdf-parse
- **AI 能力**: 豆包 lite (doubao-seed-2-0-lite-260215)

## API 路由

### POST /api/parse-document
上传 PDF 文档，返回识别的章节结构

### POST /api/generate-notes
批量生成笔记和练习题

### POST /api/stream-notes
流式生成笔记（SSE 协议）

## 备考阶段策略

| 阶段 | 距离考试 | 输出特点 | Temperature |
|------|----------|----------|-------------|
| 系统期 | > 21天 | 全面详细，建立完整框架 | 0.7 |
| 冲刺期 | 7-21天 | 重点突出，高频考点 | 0.6 |
| 急救期 | 3-7天 | 核心概念，精简提炼 | 0.5 |
| 临考期 | < 3天 | 极限压缩，必背清单 | 0.4 |

## 学习方法策略

| 方法 | 适用场景 | 输出特点 |
|------|----------|----------|
| 费曼学习法 | 深入理解概念 | 用简单语言解释，配合生活实例 |
| 康奈尔笔记法 | 系统性学习 | 主栏笔记+关键词栏+总结区 |
| 思维导图法 | 构建知识框架 | 中心主题+分支延伸+关联记忆 |
| 记忆宫殿法 | 记忆大量概念 | 具象化联想+口诀+必背清单 |
| 案例分析法 | 应用型知识 | 理论框架+真实案例+考试应用 |

## 新版工作流程

### 页面入口
- 新版笔记页面：`/notes`
- 预览页面：`/preview`

### 工作流程（5阶段）

```
┌──────────────────────────────────────────────────────────────┐
│ 阶段1: 上传 & 解析                                            │
│ - 原生代码提取原始文本（不做章节划分）                          │
│ - 支持文档上传和录音                                          │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 阶段2: AI 梳理结构                                            │
│ - API: POST /api/analyze-structure                          │
│ - 输入：原始内容 + 备考阶段                                    │
│ - 输出：章节大纲 + 每章摘要 + 整体脉络                          │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 阶段3: 用户确认 & 调整结构                                    │
│ - 组件：ChapterEditor                                        │
│ - 可增删改拖拽章节                                            │
│ - 可编辑章节标题和摘要                                        │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 阶段4: 生成详细笔记                                            │
│ - API: POST /api/generate-notes-v2                          │
│ - 按章节逐章生成                                               │
│ - 根据备考阶段调整深度                                         │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 阶段5: 生成练习题                                             │
│ - API: POST /api/generate-notes-v2 (type=questions)        │
│ - 输出格式：题目 + 【答案】+ 【解析】                          │
│ - 解析器：src/lib/question-parser.ts                         │
└──────────────────────────────────────────────────────────────┘
```

### 核心 API

| API | 功能 | 路径 |
|-----|------|------|
| 结构梳理 | AI 分析文档结构 | `POST /api/analyze-structure` |

## AI 工作流架构

### 核心文件
- `src/lib/workflow-config.ts` - 工作流配置定义

### 工作流设计
采用两阶段工作流，确保输出质量：

```
┌─────────────────────────────────────────────────────────┐
│                    流式生成请求                          │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  阶段1：内容分析（可选，默认启用）                        │
│  - 识别核心概念                                          │
│  - 提取高频考点                                          │
│  - 标注易错点                                            │
│  - 低温度 (0.3) 确保稳定性                              │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  阶段2：内容生成                                        │
│  - 基于分析结果生成笔记/练习题                            │
│  - 使用学习方法模板                                      │
│  - 备考阶段自适应温度                                    │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  阶段3：质量检查                                        │
│  - 格式完整性检查                                        │
│  - 标记合规性检查                                        │
│  - LaTeX 公式检查                                        │
└─────────────────────────────────────────────────────────┘
```

### 质量控制机制
| 检查项 | 要求 | 缺失时的建议 |
|--------|------|-------------|
| 重点标记 | 必须包含 【重点】 或 【核心】 | 添加考试高频考点标记 |
| LaTeX 公式 | 涉及公式时必须使用 $...$ 格式 | 转换为 LaTeX 格式 |
| 关键词 | 笔记应包含关键词总结 | 用 \| 分隔列出关键词 |
| 选择题 | 必须有 A/B/C/D 选项 | 检查题目格式 |
| 填空题/简答题 | 必须包含对应题型 | 验证题目数量 |

### 内容标记体系
| 标记 | 含义 | 颜色 |
|------|------|------|
| 【重点】 | 考试高频考点 | highlight |
| 【核心】 | 必须掌握的核心概念 | important |
| 【考点】 | 历年真题相关 | accent |
| 【难点】 | 容易混淆或出错 | note |
| 【注意】 | 需要特别关注 | note |
| 【易错】 | 常见错误点 | important |
| 【警示】 | 考试陷阱 | important |
